## Why

为了拓展知识库场景的使用范围，需要支持从网页地址导入内容。大量知识以网页形态存在，当前仅支持文件导入会导致收集与维护成本高、可用内容受限。

## What Changes

- 导入流程支持新增“网页地址（URL）”作为数据来源，可配置要抓取的网页地址以导入网页内容
- 导入后的网页内容进入与文件一致的解析、分段、嵌入与索引流程，使其可参与混合搜索（向量 + 全文）
- 搜索结果与文档列表在 UI 上区分“网页”与“文件”类型，并提供与网页相关的展示信息（如标题、URL）与交互（如打开链接）

## Capabilities

### New Capabilities

- `web-url-import`: 支持配置网页地址作为数据源并抓取网页内容导入知识库
- `web-document-search`: 网页类型数据可被全文/向量/混合搜索召回与展示
- `document-type-ui`: 在 UI 中区分网页与文件类型并在列表/结果中正确展示与交互

## Impact

- Main 进程：新增网页抓取、HTML 解析与正文提取能力，并接入现有导入管线
- 数据库与索引：为网页文档增加必要元数据字段（如 url、title、来源类型），确保全文索引与向量索引覆盖网页内容
- 搜索：统一召回与排序逻辑，确保网页类型在搜索结果中可区分且可过滤/可交互
- Renderer：导入入口新增 URL 配置界面，搜索结果与文档列表展示“网页/文件”类型差异
