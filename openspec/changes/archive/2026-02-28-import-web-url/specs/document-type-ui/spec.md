## ADDED Requirements

### Requirement: 文档列表区分网页与文件类型
系统 MUST 在文档列表中区分并展示文档来源类型，至少包含：网页（web）与文件（file）。

#### Scenario: 列表同时存在网页与文件
- **WHEN** 用户打开包含网页与文件混合的知识库文档列表
- **THEN** 系统 MUST 对每条文档展示其来源类型，使用户可直观区分网页与文件

### Requirement: 网页类型文档展示标题与 URL
系统 MUST 在网页类型文档的列表项与详情信息中展示网页相关信息，至少包含：标题（若可用）与 URL。

#### Scenario: 展示网页来源信息
- **WHEN** 用户查看一个网页类型文档的列表项或详情
- **THEN** 系统 MUST 展示该文档的 url，且在可用时展示 title

### Requirement: 搜索结果区分网页与文件并提供对应交互
系统 MUST 在搜索结果中区分网页与文件类型，并为网页类型结果提供“打开链接”的交互入口。

#### Scenario: 从搜索结果打开网页
- **WHEN** 用户在搜索结果中点击某条网页类型结果的“打开链接”
- **THEN** 系统 MUST 打开该结果对应的 url

### Requirement: 支持在 UI 上按类型筛选或聚合查看
系统 MUST 支持在 UI 上对网页与文件进行筛选或聚合查看，至少提供与搜索能力一致的过滤维度：仅文件、仅网页、全部。

#### Scenario: 仅查看文件文档
- **WHEN** 用户在 UI 选择“仅文件”的过滤条件
- **THEN** 系统 MUST 仅展示文件类型的文档或搜索结果
